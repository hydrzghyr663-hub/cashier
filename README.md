# نظام كاشير عربي

هذا مشروع بسيط لنظام نقطة بيع باستخدام Python وSQLite وHTML/CSS بدون أي مكتبات خارجية إضافية.

## المميزات الأساسية
- ✅ إضافة أصناف جديدة وتعديلها وحذفها
- ✅ إدارة المخزون تلقائياً
- ✅ تسجيل عمليات البيع مع الأرباح
- ✅ حفظ البيانات في قاعدة SQLite

## المميزات الجديدة - نظام إدارة الزبائن 🎉
- ✅ إضافة وتعديل وحذف الزبائن
- ✅ تخزين بيانات الزبون (الاسم، الهاتف، البريد، العنوان)
- ✅ تتبع سجل شراء كل زبون
- ✅ عرض إجمالي المشتريات لكل زبون
- ✅ واجهة بتبويبات: المنتجات | الزبائن | المبيعات
- ✅ إحصائيات حية للزبائن والمبيعات

## المميزات المتقدمة - تتبع طريقة الدفع 💰
- ✅ تحديد طريقة الدفع لكل عملية: **نقدي (💵)** أو **دين (🔴)**
- ✅ اختيار الزبون مباشرة من نموذج البيع
- ✅ تقسيم المشتريات حسب طريقة الدفع في تفاصيل الزبون
- ✅ عرض ملخص مالي يوضح:
  - إجمالي المشتريات
  - إجمالي المبيعات النقدية (💵)
  - إجمالي الديون المتبقية (🔴)
  - عدد عمليات الشراء
- ✅ سجل مفصل لكل عملية مع نوع الدفع واللون المميز

## التشغيل
1. افتح المجلد في الطرفية.
2. أنشئ بيئة افتراضية:
   ```bash
   python -m venv .venv
   ```
3. فعّل البيئة:
   ```bash
   .venv\Scripts\activate
   ```
4. شغّل التطبيق:
   ```bash
   python app.py
   ```
5. افتح المتصفح على:
   ```text
   http://127.0.0.1:5000
   ```

## النشر برابط عام (GitHub + Render)

### 1) رفع المشروع إلى GitHub

إذا كان Git مثبتًا على جهازك:

```bash
git init
git add .
git commit -m "Initial POS system"
git branch -M main
git remote add origin https://github.com/<USERNAME>/<REPO>.git
git push -u origin main
```

إذا كان Git غير مثبت:

1. أنشئ مستودعًا جديدًا في GitHub.
2. اختر Upload files.
3. اسحب كل ملفات المشروع (عدا `pos.db` لأنه محلي).
4. اضغط Commit changes.

### 2) نشره على Render

1. ادخل إلى Render وسجّل الدخول.
2. اختر New + ثم Web Service.
3. اربط مستودع GitHub الخاص بالمشروع.
4. Render سيقرأ `render.yaml` تلقائيًا.
5. أضفتُ إعدادًا لتخزين SQLite على مسار دائم (`POS_DB_PATH=/var/data/pos.db`) حتى لا تضيع البيانات عند إعادة التشغيل.
6. بعد اكتمال النشر ستحصل على رابط عام مثل:
   `https://cashier-system.onrender.com`

### 3) إرسال الرابط للزبون

أرسل للزبون رابط Render فقط، وسيتمكن من فتح النظام مباشرة بدون تثبيت أي شيء.

## قاعدة البيانات

### الجداول الرئيسية

#### جدول `customers` (جديد)
```sql
id          - معرّف فريد
name        - اسم الزبون
phone       - رقم الهاتف (فريد)
email       - البريد الإلكتروني
address     - العنوان
created_at  - تاريخ التسجيل
```

#### جدول `items`
```sql
id          - معرّف فريد
name        - اسم المنتج
buy_price   - سعر الشراء
sell_price  - سعر البيع
stock       - كمية المخزون
```

#### جدول `sales`
```sql
id              - معرّف فريد
item_id         - معرّف المنتج
customer_id     - معرّف الزبون (اختياري - جديد)
quantity        - الكمية
buy_total       - إجمالي الشراء
sell_total      - إجمالي البيع
profit          - الربح
sold_at         - تاريخ البيع
```

## الميزات التي تم إضافتها

### واجهة المستخدم
- **نظام التبويبات**: تبويبات سهلة للتنقل بين المنتجات والزبائن والمبيعات
- **بطاقات الزبائن**: عرض منظم لبيانات كل زبون مع أزرار التحكم
- **إحصائيات حية**: عرض الزبائن المسجلين والأرباح

### الوظائف الجديدة
- `list_customers()` - عرض جميع الزبائن
- `get_customer()` - الحصول على بيانات زبون معين
- `get_customer_sales()` - عرض سجل شراء الزبون
- `add_customer()` - إضافة زبون جديد
- `update_customer()` - تحديث بيانات الزبون
- `delete_customer()` - حذف الزبون

### المسارات الجديدة
- `POST /customers` - إضافة زبون جديد
- `POST /customers/{id}/update` - تحديث الزبون
- `POST /customers/{id}/delete` - حذف الزبون
- `GET /customers/{id}/details` - عرض تفاصيل الزبون

## أمثلة الاستخدام

### إضافة زبون عبر الواجهة
1. انقر على تبويب "الزبائن"
2. أدخل البيانات (الاسم، الهاتف، البريد، العنوان)
3. انقر "إضافة زبون"

### تحديث بيانات الزبون
1. اذهب لتبويب "الزبائن"
2. عدّل البيانات في النموذج أسفل البطاقة
3. انقر "تحديث"

### عرض سجل الشراء
1. اذهب لتبويب "الزبائن"
2. انقر "التفاصيل" على بطاقة الزبون
3. اعرض جميع عمليات الشراء والإجمالي

## المتطلبات

- Python 3.6+

- الواجهة مدعومة بنسبة 100% باللغة العربية
- الكود مكتوب بطريقة بسيطة وسهلة الفهم
