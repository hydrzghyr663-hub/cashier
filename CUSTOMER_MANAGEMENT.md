# نظام إدارة الزبائن - Customer Management System

## نظرة عامة
تم إضافة نظام إدارة زبائن متكامل إلى تطبيق نقطة البيع (POS). يوفر النظام إمكانيات شاملة لإدارة بيانات الزبائن وتتبع سجل شرائهم.

---

## المميزات الرئيسية

### 1. **إدارة بيانات الزبائن**
- ✅ إضافة زبائن جدد بسهولة
- ✅ تعديل بيانات الزبائن الحالية
- ✅ حذف الزبائن من النظام
- ✅ عرض قائمة كاملة بجميع الزبائن

### 2. **بيانات الزبون**
كل زبون يحتفظ بـ:
- **الاسم** (مطلوب)
- **رقم الهاتف** (مطلوب - فريد)
- **البريد الإلكتروني** (اختياري)
- **العنوان** (اختياري)
- **تاريخ التسجيل** (تلقائي)

### 3. **سجل الشراء**
- ✅ ربط كل عملية بيع بالزبون
- ✅ عرض سجل شراء الزبون الكامل
- ✅ عرض إجمالي المبيعات لكل زبون
- ✅ تتبع تاريخ كل عملية

### 4. **الواجهة الرسومية**
- **نظام التبويبات**: المنتجات | الزبائن | المبيعات
- **بطاقات الزبائن**: عرض منظم لبيانات الزبون مع أزرار التعديل والحذف
- **نماذج سهلة الاستخدام**: إضافة وتحديث البيانات بسهولة
- **إحصائيات حية**: عدد الزبائن المسجلين والمبيعات والأرباح

## التغييرات في قاعدة البيانات

### جدول جديد: `customers`
```sql
CREATE TABLE customers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    phone TEXT NOT NULL UNIQUE,
    email TEXT,
    address TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

### تعديل الجدول: `sales`
تمت إضافة عمود جديد:
```sql
ALTER TABLE sales ADD COLUMN customer_id INTEGER
```

## الوظائف المضافة

### وظائف إدارة الزبائن

#### `list_customers(db_path=DB_PATH)`
- **الوصف**: الحصول على قائمة بجميع الزبائن
- **الإرجاع**: قائمة بجميع الزبائن مع بياناتهم

#### `get_customer(customer_id, db_path=DB_PATH)`
- **الوصف**: الحصول على بيانات زبون معين
- **المدخلات**: معرّف الزبون
- **الإرجاع**: بيانات الزبون

#### `get_customer_sales(customer_id, db_path=DB_PATH)`
- **الوصف**: الحصول على سجل شراء الزبون
- **المدخلات**: معرّف الزبون
- **الإرجاع**: قائمة بجميع عمليات الشراء للزبون

#### `add_customer(name, phone, email=None, address=None, db_path=DB_PATH)`
- **الوصف**: إضافة زبون جديد
- **المدخلات**: الاسم، رقم الهاتف، البريد (اختياري)، العنوان (اختياري)
- **الأخطاء المحتملة**: الاسم أو الهاتف مفقود، رقم الهاتف مسجل بالفعل

#### `update_customer(customer_id, name, phone, email=None, address=None, db_path=DB_PATH)`
- **الوصف**: تحديث بيانات الزبون
- **المدخلات**: معرّف الزبون، والبيانات الجديدة
- **الأخطاء المحتملة**: نفس أخطاء الإضافة

#### `delete_customer(customer_id, db_path=DB_PATH)`
- **الوصف**: حذف الزبون من النظام
- **المدخلات**: معرّف الزبون
- **ملاحظة**: لا يتم حذف سجلات الشراء

## المسارات (Routes) الجديدة

### POST Routes

#### `POST /customers`
- **الوصف**: إضافة زبون جديد
- **المدخلات**: `name`, `phone`, `email` (اختياري), `address` (اختياري)
- **الاستجابة**: إعادة توجيه مع رسالة النجاح/الخطأ

#### `POST /customers/{id}/update`
- **الوصف**: تحديث بيانات الزبون
- **المدخلات**: نفس مدخلات الإضافة
- **الاستجابة**: إعادة توجيه مع رسالة النجاح/الخطأ

#### `POST /customers/{id}/delete`
- **الوصف**: حذف الزبون
- **الاستجابة**: إعادة توجيه مع رسالة النجاح

### GET Routes

#### `GET /customers/{id}/details` (مستقبلي)
- **الوصف**: عرض تفاصيل الزبون وسجل شرائه
- **الاستجابة**: صفحة HTML بتفاصيل الزبون
